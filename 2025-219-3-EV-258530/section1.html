How I met the Basic Requirements

BR1: Collect and prepare the data
a)Retrieved data from Kaggle, which had been taken from the FAA
b)Read in CSV file and cleaned it to store just the breed names and the total strikes for each breed in a separate CSV file

BR2: Data analytics and visualisation
a)Retrieved data from Kaggle, which had been taken from the FAA
b)Graphs are labelled in MatPlotLib on X and Y axes and titled for bar charts and each segment named in the case of a pie chart c)Ordered lists are used so that graphs can be made (ie 2 lists like strikes and breeds sharing index positions to make access simple with a for loop)

BR3: Create a basic interactive information system interface
a)HTML Page
b)graphs are created in python and displayed with Matplotlib. These are saved as screenshots and displayed as static images on the webpage


How I met the Advanced Requirements

AR1: Interactive Data
AR HTML Page contains text boxes to input values that get sent to python to update the graphs and thus filters them. This allows direct user interaction with the data visualisations. (Also SVG graph for strikes per state allows users to hover over the data to see exact values)

AR2: Form/Poll
Situation Designer: Allows users to fill out and submit a form in which they create a bird (choose its name, breed and size ) and choose the conditions on a particular day. a) The name, size and breed are taken as a string, the weather is taken as a boolean (cloudy/not) and operating airlines on that day (list)
b)Data is stored in a realtime database (Firebase) by a javascript function triggered by the submit button
c)All previous scenarios are displayed on the webpage along with their respective probabilities of occuring based on the datase

AR3: Reccomendation system
Details about various airlines is shown on webpage and reccomends certain ones based on safety
