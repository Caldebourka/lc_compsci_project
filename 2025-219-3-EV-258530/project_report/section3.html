Basic Requirements

Static graph appearance on html webpage. All graphs are pregenerated with matplotlib library and saved as .pngs using the save. Code runs in python to generate graphs and each can be individually generated through a menu tree in the shell

Advanced Requirements

Multi-page layout in html. One page for the form, allowing users to select details and submit a bird of their creation (incl string: name breed and size, integer: boolean:iscloudy?). This data is then collected and sent to firebase for it to be accessed in python for statistics to be run on it. The chance it will be in a strike is then graphed with pygal and displayed on the webpage along with previously created birds in the database (referred to as bird maker). The other page then contains graphs pertaining to the dataset as a whole. Some may be filtered with a textbox allowing the user to input the number of bird breeds they wish to see from an ordered list of the most common breeds in strikes for example. NB python code (html_interface.py) must be running before and during html at all times. This is because of the event listener built into the python code that checks for updates in the database and as a result is responsible for updating the graphs when a new value for n is inputted.
